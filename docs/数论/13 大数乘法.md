

## 一、什么是大数乘法问题 

**大数乘法**通常不是指数学意义上的“超长整数相乘”，而是指：

> **在程序中，`a * b` 这个运算本身可能会溢出，尤其是在取模运算中。**

典型场景：

- `a, b ≤ 10^18`
- 要计算 `(a * b) % p`
- `p ≤ 10^18`

此时哪怕用 `long long`，`a * b` 也已经炸锅了。

------

## 二、溢出问题的本质 

### 1. int 溢出还能救

```cpp
int a = 50000, b = 50000;
int ans = a * b;        // 溢出
long long ok = 1LL * a * b;  // 正确
```

结论：
👉 **int 不够，用 long long**

------

### 2. long long 也会溢出

```cpp
long long a = 1e18, b = 1e18;
long long p = 1e18 + 7;

long long x = (a * b) % p;   // 已经溢出了
```

这时问题升级：👉 **乘法本身做不出来**

------

## 三、慢速乘（加法模拟乘法）

### 核心思想

把乘法拆成加法，永远不让中间结果超过 `p`。

利用二进制分解：
$$a \times b = a \times (b_0·2^0 + b_1·2^1 + \dots)$$

------

### 算法描述

- 类似快速幂
- 每次：
  - 如果 `b` 的最低位是 1，加上当前的 `a`
  - `a = a * 2 % p`
  - `b >>= 1`

------

### 代码示例（OI 标准写法）

```cpp
long long mul(long long a, long long b, long long p) {
    long long res = 0;
    while (b) {
        if (b & 1) res = (res + a) % p;
        a = (a + a) % p;
        b >>= 1;
    }
    return res;
}
```

------

### 举个数值例子

计算：

```cpp
13 × 11 % 100
```

二进制分解：

```cpp
11 = 1011₂ = 8 + 2 + 1
```

过程：

- res += 13
- res += 13×2
- res += 13×8

全部通过“加法 + 取模”完成。

------

### 特点总结

- ✅ 绝对安全
- ❌ 比普通乘法慢
- ⏱ 复杂度：`O(log b)`

------

### 四、快速乘（浮点数 trick）⚡

#### 核心思想

利用公式：
$$(a \times b) \bmod p = a \times b - \left\lfloor \frac{a \times b}{p} \right\rfloor \times p$$

直接算 `a*b` 会溢出，但：

> **`a*b/p` 的结果可以用浮点数近似表示**

------

#### 实现思路

- 用 `long double` 计算 `a*b/p`
- 转成整数 `k`
- 用减法修正结果

------

#### 代码示例

```cpp
long long mul(long long a, long long b, long long p) {
    long long k = (long double)a * b / p;
    long long res = a * b - k * p;
    res %= p;
    if (res < 0) res += p;
    return res;
}
```

------

#### 使用场景

- `a, b, p ≤ 10^18`
- 对性能要求高
- 评测环境浮点误差可控（OI 常见）

------

#### 特点总结

- 🚀 非常快
- ⚠️ 有浮点误差风险
- 🧠 属于“竞赛技巧型方法”

------

### 五、三种方法对比 

| 方法   | 是否溢出 | 速度 | 可靠性 | 常见用途        |
| ------ | -------- | ---- | ------ | --------------- |
| 直接乘 | 可能     | 最快 | 低     | 小数据          |
| 慢速乘 | 不会     | 较慢 | 极高   | 教学 / 保守写法 |
| 快速乘 | 不会     | 很快 | 中高   | NOI / 大模数    |



## 演示-乘法溢出问题 

### `一、实验目标`

输入正整数a b，计算a*b mod p的结果，p为1e6。

`输入样例`：

```cpp
1000001 1000002
```

`输出样例`：

```cpp
2
```

`数据范围`：

- 100%：1<=a、b<=10^7

### `二、分析`

#### 1、思路

如果用int来定义a、b，并且直接算a*b % p ，由于a*b超出了int的范围，导致结果溢出。

需要先转换为long long 进行计算，算好以后求模，再转换回int就可以了。

```c
#include <iostream>
using namespace std;
const int MOD=1e6;
int main() {
  int a, b;
  cin>>a>>b;
  //cout<<(a*b)%MOD<<endl;
  cout<<(1LL*a*b)%MOD<<endl;
  return 0;
}

```



## 演示-慢速乘模板 

### `一、实验目标`

两个正整数a和b，求a*b，由于数很大，输出其对1e15求模。

`注意`：

- 1<= a、b <=10^18
- a*b可能会超出long long 的长度。

`输入样例`（a=1e15+1e14, b=1e10）：

```cpp
1100000000000000 10000000000
```

`输出样例`：

```cpp
0
```

### `二、分析`

#### 1、思路

假设 b = 2^xn + 2^xn-1 + ... + 2^x0，则：

a*b = a* (2^xn + 2^xn-1 + ... 2^x0)

`思路`－循环b的二进制拆分：

- 每次 ans += a，取模
- a *= 2 取模, b/=2

`复杂度`：O(logn)， 也可以考虑用大一点的底提高性能。





### 一、核心问题分析

- 给定

```cpp
  1 ≤ a, b ≤ 10^18
  ```

- 直接算 `a * b` 会**溢出 `long long`**

- 只要求结果

```cpp
  (a * b) mod 1e15
  ```

👉 重点：**模数是 1e15，不是质数，也不能用逆元**

------

### 二、解题思路（慢速乘的灵魂）

#### 1️⃣ 二进制拆分思想

任何整数 `b` 都可以写成：

```cpp
b = 2^x0 + 2^x1 + ... + 2^xk
```

于是：

```cpp
a * b
= a*(2^x0 + 2^x1 + ...)
= a*2^x0 + a*2^x1 + ...
```

而 `a * 2^x`，可以通过**不断翻倍**得到。

------

#### 2️⃣ 把「乘法」变成「加法 + 翻倍」

循环做三件事：

- 如果 `b` 的最低位是 1
   👉 把当前的 `a` 加进答案
- `a = a * 2 (mod MOD)`
- `b = b / 2`

直到 `b == 0`

整个过程只做 **加法 + 位运算**，完全不溢出。

------

#### 3️⃣ 复杂度

- `b` 每次除以 2
- 循环次数：`O(log b)`，最多 ~60 次
- 对 10¹⁸ 级别输入，轻松通过

```c
#include <iostream>
using namespace std;
typedef long long ll;
const ll MOD=1e15;

ll multi(ll a, ll b) {
  ll ans=0;
  while(b) {
    if(b%2==1)
      ans = (ans+a)%MOD;
    a = (a+a)%MOD;
    b /= 2;
  }
  return ans;
}

int main() {
  ll a, b;
  cin>>a>>b;
  
  cout<<multi(a, b)<<endl;
  return 0;
}

```



## 演示-快速乘模板 

### `一、实验目标`

两个正整数a和b，求a*b，由于数很大，输出其对1e15求模。

`注意`：

- 1<= a、b <=10^18
- a*b可能会超出long long 的长度。

`输入样例`（a=1e15+1e14, b=1e10）：

```cpp
1100000000000000 10000000000
```

`输出样例`：

```cpp
0
```

### `二、分析`

#### 1、思路

x = x/p∗p + x%p => x%p = x - x/p∗p

假设 x = a∗b，代进上面的公式：(a∗b)%p = a∗b - a∗b/p∗p

对于两个大数的差，如果两个数都溢出，差也不会变。

但问题出在a∗b先溢出然后/p结果就不对了，因此先用long double来计算a/p∗b，然后再将之取整再计算差。

最后，如果结果为负就加一下MOD（溢出的后果）。

`复杂度`：O(1)



```c
#include <iostream>
using namespace std;
typedef long long ll;
const ll MOD=1e15;
int main() {
  ll a, b;
  cin>>a>>b;
  //cout<<((a*b)%MOD)<<endl;
  //cout<<((a%MOD)*(b%MOD)%MOD)<<endl;  
  
  a %= MOD, b %= MOD;   
  long double z = (long double)a/MOD*b;
  ll z2 = z;
  ll r = a*b - z2*MOD;
  if(r<0) r += MOD;
  cout<<r<<endl;
  
  return 0;
}

```



## 练习-乘积求余 

### `题目描述`

输入n和m，求[1, n]内每个数的m次方的乘积，结果对10^15+7求余。

`输入样例1`：

```cpp
3 2
```

`输出样例1`：

```cpp
36
```

样例1说明：1^2 * 2^2 * 3^2 = 4*9 = 36

`输入样例2`：

```cpp
10 10
```

`输出样例2`：

```cpp
481705703325804
```

`数据范围`：

- 1<=n、m<=50



```c
#include <iostream>
using namespace std;
typedef long long ll;
const ll P = 1e15+7;

ll mul(ll a, ll b) {
  /* 快速乘
  a %= P, b %= P;   
  long double z = (long double)a/P*b;
  ll z2 = z;
  ll r = a*b - z2*P;
  if(r<0) r += P;
  return r;
  */
  //慢速乘
  ll ans=0;
  while(b) {
    if(b%2==1)
      ans = (ans+a)%P;
    a = (a+a)%P;
    b /= 2;
  }
  return ans;
}

ll pow(ll a, ll b) {
  long long ans=1;
  while(b) {
    if(b%2==1)
      ans = mul(ans, a);
    a = mul(a, a);
    b /= 2;
  }
  return ans;
}

int main() {
  int n, m;
  cin>>n>>m;
  ll ans = 1;
  for(int i=1; i<=n; i++) {
    ll x = pow(i, m);
    ans = mul(ans, x);
  }
  cout<<ans<<endl;
  return 0;
}


```
