

## 一、中国剩余定理（CRT）是什么？

### 1️⃣ 标准表述（互质版）

给定同余方程组：
[
$$\begin{cases}
x \equiv a_1 \pmod{m_1}   \ x \equiv a_2 \pmod{m_2}\
\vdots\
x \equiv a_k \pmod{m_k}
\end{cases}$$
]

若
[
$$\gcd(m_i, m_j)=1 \quad (i\ne j)$$
]

则：

- **一定有解**
- 解在模$$M = m_1m_2\cdots m_k$$ 下唯一

------

## 二、CRT 的“存在性根源”

你已经走过这条链：

> 裴蜀定理 → 扩欧 → 逆元 → 同余方程

CRT 本质上是：

- 多个同余方程
- 通过 **逆元 + 裴蜀定理**
- 拼成一个更大的线性同余

没有新工具，只是工具的“群体协作”。🤝

------

## 三、两模版本（最常考）

[
\begin{cases}
x \equiv a \pmod m\
x \equiv b \pmod n
\end{cases}
]

### 情况 1：(\gcd(m,n)=1)

✔ 一定有解
✔ 解模 (mn) 唯一

------

### 情况 2：(\gcd(m,n)=d>1)

- 有解 ⇔ (a \equiv b \pmod d)
- 有解时可合并成一个模为
  [
  \text{lcm}(m,n)
  ]

👉 这就是 **扩展 CRT**

------

## 四、CRT 的构造思路（互质版）

设：
[
M = \prod m_i,\quad M_i = \frac{M}{m_i}
]

找：
[
t_i \equiv M_i^{-1} \pmod{m_i}
]

最终解：
[
x \equiv \sum a_i M_i t_i \pmod M
]

每一项都只在“自己的模数”下生效，其它模数下自动归零。
像给每个模数发一张**专属通行证**。🎫

------

## 五、一个完整例子

[
\begin{cases}
x \equiv 2 \pmod 3\
x \equiv 3 \pmod 5\
x \equiv 2 \pmod 7
\end{cases}
]

[
M=105
]

[
\begin{aligned}
M_1&=35,\quad 35^{-1}\equiv 2\pmod3\
M_2&=21,\quad 21^{-1}\equiv 1\pmod5\
M_3&=15,\quad 15^{-1}\equiv 1\pmod7
\end{aligned}
]

[
x \equiv 2·35·2 + 3·21·1 + 2·15·1 = 233 \equiv 23 \pmod{105}
]

## 六、CRT 在竞赛中的作用

### 1️⃣ 合并多个限制

- 周期问题
- 时间同步
- 取模条件叠加

------

### 2️⃣ 加速大数运算

把一个大模数：
[
\bmod (p_1p_2)
]

拆成：
[
\bmod p_1,\quad \bmod p_2
]

算完再 CRT 合并。
这是高精度与密码学的常用技巧。🔐

------

### 3️⃣ 破解“看似不可能”的同余题

- 多模数
- 不同余条件
- 模数不互质

👉 90% 的解法都在 CRT 或扩展 CRT 里

## 一、互质 CRT 模板（最经典）

适用条件：
[
\gcd(m_i, m_j)=1
]



### 思路一句话

> 用逆元把每个模数的“声音”单独放大，再叠加。

------

### 代码模板（C++）

```cpp
using ll = long long;

// 扩展欧几里得
ll exgcd(ll a, ll b, ll &x, ll &y) {
    if(b == 0) {
        x = 1; y = 0;
        return a;
    }
    ll x1, y1;
    ll g = exgcd(b, a % b, x1, y1);
    x = y1;
    y = x1 - (a / b) * y1;
    return g;
}

// 互质 CRT
// a[i] ≡ x (mod m[i]), i = 1..n
ll CRT(int n, ll a[], ll m[]) {
    ll M = 1;
    for(int i = 1; i <= n; i++) M *= m[i];

    ll ans = 0;
    for(int i = 1; i <= n; i++) {
        ll Mi = M / m[i];
        ll x, y;
        exgcd(Mi, m[i], x, y);   // x = Mi^{-1} mod m[i]
        x = (x % m[i] + m[i]) % m[i];
        ans = (ans + a[i] * Mi % M * x % M) % M;
    }
    return (ans % M + M) % M;
}
```

## 二、扩展 CRT（模数不互质）

适用条件：
[
\gcd(m_i, m_j) \ne 1
]

这是 **竞赛高频考点**。

------

### 两个方程合并模板

合并：
[
\begin{cases}
x \equiv a_1 \pmod{m_1}\
x \equiv a_2 \pmod{m_2}
\end{cases}
]

------

### 代码模板（exCRT）

```cpp
using ll = long long;

// 扩展欧几里得
ll exgcd(ll a, ll b, ll &x, ll &y) {
    if(b == 0) {
        x = 1; y = 0;
        return a;
    }
    ll x1, y1;
    ll g = exgcd(b, a % b, x1, y1);
    x = y1;
    y = x1 - (a / b) * y1;
    return g;
}

// 合并两个同余方程
// x ≡ a1 (mod m1)
// x ≡ a2 (mod m2)
// 返回是否有解，结果存入 a1, m1
bool merge(ll &a1, ll &m1, ll a2, ll m2) {
    ll x, y;
    ll g = exgcd(m1, m2, x, y);
    if((a2 - a1) % g != 0) return false;

    ll lcm = m1 / g * m2;
    ll t = (a2 - a1) / g;
    x = (x % (m2 / g) + (m2 / g)) % (m2 / g);

    ll k = t % (m2 / g) * x % (m2 / g);
    a1 = (a1 + m1 * k) % lcm;
    m1 = lcm;
    a1 = (a1 % m1 + m1) % m1;
    return true;
}
```

------

### 多个方程用法

```cpp
ll a = a[1], m = m[1];
for(int i = 2; i <= n; i++) {
    if(!merge(a, m, a[i], m[i])) {
        // 无解
    }
}
// 最终解：x ≡ a (mod m)
```

## 三、使用时的“肌肉记忆”口诀 

- **模数互质** → CRT（逆元版）
- **模数不互质** → 扩展 CRT（合并版）
- **是否有解** → 看 `(a2 - a1) % gcd == 0`
