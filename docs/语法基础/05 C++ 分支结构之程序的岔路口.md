# C++ 炼气期之分支结构：程序的岔路口

## 1. 前言

程序是处理数据的流水线。一条流水线是如何工作的呢？通常是先输入数据，然后处理，最后输出结果——这就是所谓的**顺序结构**。但人类的思维不会总是线性的，面对复杂世界时，我们频繁地根据条件做出判断：如果下雨就带伞，如果饿了就吃饭。

**分支结构**（也叫选择结构）正是为了让程序拥有"判断能力"而存在的语法机制。通过它，程序可以根据不同的条件选择不同的执行路径，而不是永远一成不变地走下去。

C++ 中的分支结构主要包含以下几类：

- if 语句
- if-else 语句
- else-if 链
- switch 语句
- 三目运算符

---

## 2. if 语句

### 2.1 基本形式

if 语句是最基础的分支结构，它的语法直观明了：

```c++
if (条件表达式) {
    // 条件为 true 时执行的代码
}
```





这里的**条件表达式**可以是任何能计算出布尔值（true 或 false）的表达式。在 C++ 中，0 被视为 false，任何非 0 值都被视为 true。

> **Tip：** 如果条件表达式后的代码块只有一条语句，可以省略花括号，但为了代码可读性和后期维护，建议始终使用花括号。

先从一个简单的示例开始：

```c++
#include <iostream>
using namespace std;

int main() {
    int score = 85;
    if (score >= 60) {
        cout << "恭喜，考试及格！" << endl;
    }
    return 0;
}
```



当 score >= 60 为 true 时，控制台会输出"恭喜，考试及格！"；如果分数低于 60，则什么都不输出。

### 2.2 关系运算符

分支结构的核心离不开关系运算，常见的关系运算符如下：

| 运算符 | 含义   | 示例      |
|-----|------|---------|
| ==  | 等于   | a == b  |
| !=  | 不等于  | a != b  |
| <   | 小于   | a < b   |
| >   | 大于   | a > b   |
| <=  | 小于等于 | a <= b  |
| >=  | 大于等于 | a >= b  |

> **注意：** == 是相等比较，而 = 是赋值操作，初学者常把两者混淆。如果写成 if (a = 5)，不会报错，但表达式的结果恒为 true，因为赋值表达式的值就是赋进去的值（这里是 5，非 0 即为 true），这是一个经典的逻辑陷阱。

## 3. if-else 语句

现实中的判断往往有"双向选择"：如果条件成立做什么，否则做什么。if-else 结构正好解决了这个问题。

```c++
if (条件表达式) {
    // 条件为 true 时执行
} else {
    // 条件为 false 时执行
}
```



下面是一个判断奇偶数的例子：

```c++
#include <iostream>
using namespace std;

int main() {
    int num = 7;
    if (num % 2 == 0) {
        cout << num << " 是偶数" << endl;
    } else {
        cout << num << " 是奇数" << endl;
    }
    return 0;
}
```

运行结果：

`
7 是奇数
`

if-else 确保了无论条件是否成立，都有一个确定的执行路径——程序不会有"悬而未决"的状态。

---

## 4. else-if 链

当判断条件不止两个时，可以使用 else-if 链来实现多路分支。

```c++
if (条件1) {
    // 条件1 成立
} else if (条件2) {
    // 条件1 不成立且条件2 成立
} else if (条件3) {
    // 条件1、2 不成立且条件3 成立
} else {
    // 以上条件均不成立
}
```



来看一个成绩等级评定的例子：

```c++
#include <iostream>
using namespace std;

int main() {
    int score = 78;
    if (score >= 90) {
        cout << "等级：A" << endl;
    } else if (score >= 80) {
        cout << "等级：B" << endl;
    } else if (score >= 70) {
        cout << "等级：C" << endl;
    } else if (score >= 60) {
        cout << "等级：D" << endl;
    } else {
        cout << "等级：E（不及格）" << endl;
    }
    return 0;
}
```

输出：

`
等级：C
`

> **Tip：** else-if 链本质上是一种嵌套的 if-else 结构的语法糖。判断会按顺序从上到下进行，一旦有一个条件满足，后续的所有条件都会被跳过。因此，条件的排列顺序会影响程序的逻辑和效率——更可能成立的条件通常应放在前面。

## 5. switch 语句

当判断的条件是基于**某个表达式的多个离散值**时，switch 语句比 else-if 链更加清晰和高效。

```c++
switch (表达式) {
    case 值1:
        // 当表达式 == 值1 时执行
        break;
    case 值2:
        // 当表达式 == 值2 时执行
        break;
    // ... 更多 case
    default:
        // 当所有 case 都不匹配时执行
        break;
}
```



一个根据星期数字输出星期名称的例子：

```c++
#include <iostream>
using namespace std;

int main() {
    int day = 3;
    switch (day) {
        case 1:
            cout << "星期一" << endl;
            break;
        case 2:
            cout << "星期二" << endl;
            break;
        case 3:
            cout << "星期三" << endl;
            break;
        case 4:
            cout << "星期四" << endl;
            break;
        case 5:
            cout << "星期五" << endl;
            break;
        case 6:
            cout << "星期六" << endl;
            break;
        case 7:
            cout << "星期日" << endl;
            break;
        default:
            cout << "无效的星期数字" << endl;
            break;
    }
    return 0;
}
```



### 5.1 break 的重要性

如果没有 break，程序会发生"穿透"（fall-through）——即执行完一个 case 后继续执行下一个 case。这在某些场景中恰恰是期望的行为，比如多个值共享同一段处理逻辑：

```c
#include <iostream>
using namespace std;

int main() {
    char grade = 'A';
    switch (grade) {
        case 'A':
        case 'B':
        case 'C':
            cout << "成绩合格" << endl;
            break;
        case 'D':
            cout << "成绩不合格" << endl;
            break;
        default:
            cout << "无效的成绩等级" << endl;
            break;
    }
    return 0;
}
```



### 5.2 switch 的限制

switch 的表达式必须是整数类型（如 int、char、enum），不能是浮点数或字符串。这是 switch 与 if 相比的局限，也是它能够在编译时优化为跳转表的原因。

| 特性       | if-else 链       | switch 语句       |
|----------|-----------------|-----------------|
| 适用场景     | 范围判断、复杂条件     | 离散值判断          |
| 数据类型     | 任意类型           | 仅整数类型（含 char）  |
| 可读性      | 条件多时不够直观       | 结构清晰、一目了然      |
| 性能       | 顺序比较           | 可优化为跳转表，性能更优   |

## 6. 三目运算符

三目运算符（? :）是 C++ 中唯一接收三个操作数的运算符，它是一种紧凑的条件表达式。

`
条件 ? 表达式1 : 表达式2
`

如果条件为 true，整个表达式的值为表达式1，否则为表达式2。

```c++
#include <iostream>
using namespace std;

int main() {
    int a = 10, b = 20;
    int maxVal = (a > b) ? a : b;
    cout << "较大值: " << maxVal << endl;
    return 0;
}
```

相当于：

```c
int maxVal;
if (a > b) {
    maxVal = a;
} else {
    maxVal = b;
}
```



> **Tip：** 三目运算符可以在需要返回值的表达式中使用，例如初始化变量、函数参数、输出语句中等。但要注意不要过度嵌套，否则会严重降低代码的可读性。

## 7. 常见陷阱与最佳实践

### 7.1 悬浮 else

C++ 中 else 会与最近的未配对的 if 结合，这就是"悬浮 else"问题：

```c
if (x > 0)
    if (y > 0)
        cout << "x 和 y 都为正数" << endl;
else
    cout << "x 为非正数" << endl;  // 本意是匹配外层 if
```



实际上，else 匹配的是内层 if，导致逻辑错误。解决方法是始终使用花括号明确层次：

```c
if (x > 0) {
    if (y > 0) {
        cout << "x 和 y 都为正数" << endl;
    }
} else {
    cout << "x 为非正数" << endl;
}
```



### 7.2 浮点数比较

浮点数的精度问题会导致直接比较不可靠：

```c
double x = 0.1 + 0.2;
if (x == 0.3) {
    // 这个条件很可能不成立
}
```

正确的做法是使用容差比较：

```c
double x = 0.1 + 0.2;
double epsilon = 1e-9;
if (abs(x - 0.3) < epsilon) {
    cout << "x 约等于 0.3" << endl;
}
```



## 8. 总结

分支结构是程序脱离"线性执行"的第一步，也是赋予程序"智能"的基础工具。C++ 提供了多种分支机制：

- if 语句：基础单路判断
- if-else 语句：双路选择
- else-if 链：多路分支
- switch 语句：离散值多路分支（性能更优）
- 三目运算符：简洁的条件表达式

选择哪种分支结构取决于具体场景：判断范围或复杂逻辑用 if，判断离散常量用 switch，简单的二选一可以用三目运算符。

写好分支结构的核心不在于记住语法，而在于养成"考虑所有可能性"的思维习惯——不仅要思考条件成立时的逻辑，也要考虑条件不成立时是否要有兜底处理，这才是健壮程序的基石。
