    11 同余

    一、同余的定义

    例 1：同余的直观理解

    取

    • $$(a = 23)$$
    • $$(b = 3)$$
    • $$(m = 10)$$

    计算余数：

    • $$(23 \div 10) 余 3$$
    • $$(3 \div 10) 余 3$$

    所以

    $$23 \equiv 3 \pmod{10}$$

    再验证等价条件：

    $$23 - 3 = 20,\quad 10 \mid 20$$

    ✔ 完全符合

    $$a \equiv b \pmod m \iff m \mid (a-b)$$

    二、同余方程（代入说明）

    例 2：同余方程转化为二元一次方程

    同余方程：

    $$4x \equiv 2 \pmod 6$$

    按定义展开：

    $$4x = 6t + 2$$

    移项：

    $$4x - 6t = 2$$

    这就是一个二元一次不定方程。

    例如取 (t = 1)：

    $$4x = 8 \Rightarrow x = 2$$

    验证： $$4 \times 2 = 8,\quad 8 \div 6\text{ 余 }2$$

    ✔ 成立

    三、同余的特性（代入说明）

    3.1 基础特性

    1️⃣ 自反性

    任取

    $$a = 17,\ m = 5$$

    显然：

    $$17 \equiv 17 \pmod 5$$

    2️⃣ 对称性

    已知： $$17 \equiv 2 \pmod 5$$

    则必然：

    $$2 \equiv 17 \pmod 5$$

    因为：

    $$17 - 2 = 15,\quad 5 \mid 15$$

    3️⃣ 传递性

    已知： $$17 \equiv 2 \pmod 5$$

    $$2 \equiv 7 \pmod 5$$

    则： $$17 \equiv 7 \pmod 5$$

    验证： $$17 - 7 = 10,\quad 5 \mid 10$$

    3.2 运算特性（重点）

    1️⃣ 同加性

    已知： $$17 \equiv 2 \pmod 5$$

    两边同时加 (c = 6)： $$17 + 6 = 23,\quad 2 + 6 = 8$$

    判断： $$23 \equiv 8 \pmod 5$$

    验证： $$23 \div 5 \text{ 余 }3,\quad 8 \div 5 \text{ 余 }3$$

    ✔ 成立

    2️⃣ 同乘性（单侧）

    已知： $$17 \equiv 2 \pmod 5$$

    乘 (c = 3)： $$17 \times 3 = 51,\quad 2 \times 3 = 6$$

    判断： $$51 \equiv 6 \pmod 5$$

    验证： $$51 \div 5 \text{ 余 }1,\quad 6 \div 5 \text{ 余 }1$$

    3️⃣ 同乘性（双侧）

    已知：

    $$17 \equiv 2 \pmod 5,\quad 13 \equiv 3 \pmod 5$$

    相乘：

    $$17 \times 13 = 221$$

    $$2 \times 3 = 6$$

    判断：

    $$221 \equiv 6 \pmod 5$$

    验证：

    $$221 \div 5 \text{ 余 }1,\quad 6 \div 5 \text{ 余 }1$$

    4️⃣ 同幂性

    已知：

    $$17 \equiv 2 \pmod 5$$

    平方：

    $$17^2 = 289,\quad 2^2 = 4$$

    判断：

    $$289 \equiv 4 \pmod 5$$

    验证：

    $$289 \div 5 \text{ 余 }4$$

    5️⃣ 同除性（互质情况）

    已知：

    $$6a \equiv 6b \pmod{35}$$

    因为：

    $$(6,35) = 1$$

    可以约掉 6：

    $$a \equiv b \pmod{35}$$

    6️⃣ 同除性（非互质，推广）

    已知：

    $$6a \equiv 6b \pmod{18}$$

    因为：

    $$(6,18) = 6$$

    只能约成：

    $$a \equiv b \pmod{18/6} = \pmod 3$$

    四、大数运算应用

    1️⃣ 乘法分解

    计算： $$123456 \times 789 \bmod 1000$$

    直接算太大，拆： $$123456 \bmod 1000 = 456$$

    $$789 \bmod 1000 = 789$$ $$(456 \times 789) \bmod 1000$$

    $$456 \times 789 = 359784$$

    $$359784 \bmod 1000 = 784$$

    ✔ 原式结果也是 784

    2️⃣ 加法分解

    $$(123456 + 789012) \bmod 1000$$

    拆：

    $$123456 \bmod 1000 = 456$$

    $$789012 \bmod 1000 = 12$$

    $$(456 + 12) \bmod 1000 = 468$$

    3️⃣ 除法反例（为什么不能直接同除）

    给出的反例：

    $$(100 / 50) \bmod 20 = 2$$

    但如果“先取模”： $$100 \bmod 20 = 0$$

    $$50 \bmod 20 = 10$$

    $$0 / 10 = 0$$

    ❌ 结果不一致

    原因一句话：

    模意义下的“除法”，不是普通除法，而是乘逆元

    “逆元” 这个概念。

    1. 核心思想

    逆元，就是“能够抵消某个运算效果”的元素。

    就像在日常生活中： - 加法的逆元：一个数的相反数，比如 ( 5 ) 的加法逆元是 (-5)，因为 ( 5 + (-5) = 0 )（加法归零）。 - 乘法的逆元：一个数的倒数，比如 ( 5 ) 的乘法逆元是 ($$\frac{1}{5}$$)，因为 ( $$5 \times \frac{1}{5} = 1$$ )（乘法归一）。

    这里的 ( 0 ) 是加法的“单位元”（加了不变），( 1 ) 是乘法的“单位元”（乘了不变）。

    2. 数学定义

    设有一个运算 ($$ \ast$$ )（比如加法或乘法），它的单位元是 ( e )（运算后结果不变：( $$a \ast e = e \ast a = a )$$）。

    对于元素 ( a )，如果存在元素 ( b )，使得：

    $$a \ast b = b \ast a = e$$

    那么 ( b ) 就叫做 ( a ) 的（关于运算 ( $$\ast $$) 的）逆元。

    3. 常见例子

    (1) 实数中的逆元

    • 加法逆元：每个实数 ( a ) 都有加法逆元 (-a)。
      例如：( 3 + (-3) = 0 )。
    • 乘法逆元：除了 ( 0 ) 以外，每个实数 ( a ) 都有乘法逆元 $$( \frac{1}{a} )$$。
      例如：( $$4 \times \frac{1}{4} = 1$$ )。
      （0 没有乘法逆元，因为任何数乘 0 都不等于 1。）

    (2) 模运算中的乘法逆元（最重要）

    在 模 ( m ) 的世界里（整数 0 到 ( m-1 )），我们经常做“取模”运算。
    比如模 7 的世界是 {0, 1, 2, 3, 4, 5, 6}。

    对于数 ( a )，它的 乘法逆元 ( x ) 要满足：

    $$a \times x \equiv 1 \ (\text{mod } m)$$

    相当于在模 ( m ) 下，( x ) 扮演了“倒数”的角色。

    例子：在模 7 下：

    • 3 的乘法逆元是 5，因为$$ ( 3 \times 5 = 15 \equiv 1 \ (\text{mod } 7) )。$$
    • 2 的乘法逆元是 4，因为$$ ( 2 \times 4 = 8 \equiv 1 \ (\text{mod } 7) )。$$
    • 0 没有逆元。

    注意：在模运算里，不是每个数都有乘法逆元，必须满足 (gcd(a, m) = 1)（互质）才有。

    4. 为什么逆元重要？

    因为有了逆元，我们就可以在相应的运算系统里做“除法”。

    • 在实数中：除以 4 等于乘以 ($$\frac{1}{4}$$)。
    • 在模运算中：求 ($$ \frac{b}{a} \ (\text{mod } m)$$ ) 等于求 ( $$b \times a^{-1} \ (\text{mod } m) $$)，其中 ( $$a^{-1} $$) 是 ( a ) 的模 ( m ) 乘法逆元。

    例子：在模 7 下计算 ( $$\frac{5}{3}$$ )（即 ($$ 5 \div 3 $$) 模 7）：

    1. 先求 3 在模 7 下的逆元：上面说了是 5。
    2. 计算 ( $$5 \times 5 = 25 \equiv 4 \ (\text{mod } 7) $$)。
    3. 所以 ($$ 5 \div 3 \equiv 4 \ (\text{mod } 7)$$ )。 验证：( $$3 \times 4 = 12 \equiv 5 \ (\text{mod } 7) $$)，正确。

    5. 总结逆元的特点

    运算类型 单位元 a 的逆元 存在条件
    实数加法 0 -a 总是存在
    实数乘法 1 1/a (a≠0) a ≠ 0 时存在
    模 m 乘法 1 满足 a·x ≡ 1 (mod m) 的 x a 与 m 互质时存在

    演示-加法求余运算

    一、实验目标

    输入n，求$$1^2 + 2^2 + ... + n^2$$的结果，由于结果可能很大，输出结果对$$10^9+7$$求余。

    输入样例1：

    10000

    输出样例1：

    383332669

    数据范围：

    • 1<= n <= 100,000

    一、解题思路

    1️⃣ 直接枚举

    最直观的想法：

    $$1^2 + 2^2 + \cdots + n^2$$

    用一个循环，从 1 加到 n，每一步累加 (i^2)，同时取模。

    • 时间复杂度：O(n)
    • 在 ($$n \le 10^5$$) 的范围内，完全可以接受
    • 注意：平方和中间结果会很大，必须用 long long

    2️⃣ 为什么要“边算边取模”？

    模运算有一个非常好用的性质：

    $$(a + b) \bmod M = ((a \bmod M) + (b \bmod M)) \bmod M$$

    所以我们可以：

    • 每次算 $$(i^2)$$ 后立刻取模
    • 每次加到答案里也取模 这样中间变量不会爆炸 💥

    3️⃣ 算法流程总结

    1. 读入 n
    2. 初始化答案 ans = 0
    3. 从 i = 1 到 n
    4. 计算 i * i
    5. 累加到 ans
    6. 对 MOD = 1e9+7 取模
    7. 输出 ans

    二、参考代码

    #include <iostream>
using namespace std;

const long long MOD = 1000000007;

int main() {
    int n;
    cin >> n;

    long long ans = 0;  // 用 long long 防止中间结果溢出

    for (int i = 1; i <= n; i++) {
        // i*i 先转成 long long，避免 int 溢出
        long long sq = 1LL * i * i;

        // 累加平方，并取模
        ans = (ans + sq) % MOD;
    }

    cout << ans << endl;
    return 0;
}

    练习-同余方程

    一、实验目标

    来自noip2012 D2T1 同余方程（洛谷P1082）

    求关于x的同余方程ax ≡ 1 (mod b)的最小正整数解。

    输入样例1：

    3 10

    输出样例1：

    7

    数据范围：

    • 40%：2<=b<=1000
    • 60％：2<=b<=50,000,000
    • 100%：2<=a、b<=2,000,000,000

    二、分析

    1、暴力思路

    直接对x: [1,b]枚举，找到第一个满足(ax-1) % b ==0的输出。

    或者对y：[1, a]枚举，找到第一个满足(by+1)%a==0的y，然后计算最小的正整数x

    2、扩展欧几里得

    用扩展欧几里得求解，然后计算[1, b]区间内的x解

    • 这题本质是：求乘法逆元
    • ax ≡ 1 (mod b) ⇔ ax + by = 1
    • 大范围数据 → 必须用扩展欧几里得
    • 记住调整公式： x = (x % b + b) % b

    “乘逆元”（也称为 乘法逆元 或 模逆元）。

    1. 基本定义

    在一个代数系统中（比如整数、实数或有模运算的整数），乘逆元 指的是一个数乘以它的逆元后，结果等于该系统的乘法单位元。

    • 对于实数：
      数 ( a ) 的乘法逆元是 ( $$\frac{1}{a} )（(a \ne 0)$$），因为

    $$a \times \frac{1}{a} = 1$$

    这里的 ( 1 ) 是实数乘法的单位元。

    2. 模运算下的乘法逆元

    在 模运算 中，这个概念特别重要。设： - 模数为 ( m )（正整数） - 整数 ( a )，满足 ( $$\gcd(a, m) = 1$$ )（即 ( a ) 与 ( m ) 互质）

    那么 ( a ) 在模 ( m ) 下的乘法逆元是一个整数 ( x )，使得：

    $$a \cdot x \equiv 1 \pmod{m}$$

    记作 ( $$x \equiv a^{-1} \pmod{m}$$ )。

    3. 为什么要求 (gcd(a,m)=1)？

    如果 ($$\gcd(a, m) = d > 1$$)，那么方程
    ( $$a x \equiv 1 \ (\text{mod } m) $$) 无解，因为：

    • 左边 ( a x ) 与 ( a ) 有公因数 ( d )（因此与 ( m ) 也有公因数 ( d )）。
    • 右边 ( 1 ) 与 ( m ) 互质，不可能与左边同余（模 ( m ) 下差是 ( m ) 的倍数）。

    所以 互质 是模逆元存在的充要条件。

    4. 如何求模逆元？

    常见方法：

    (1) 扩展欧几里得算法（最通用）

    求解方程：

    $$a x + m y = 1$$

    由于 ($$\gcd(a,m)=1$$)，此方程必有整数解。
    解出的 ( x ) 就是 ( a ) 在模 ( m ) 下的逆元（通常取 ( x ) 模 ( m ) 到 ( 0 \dots m-1 ) 范围）。

    例子：求 ( 3 ) 在模 ( 11 ) 下的逆元。
    解 ( 3x + 11y = 1 )，得 ( x = 4 )（因为 ( $$3 \times 4 = 12 \equiv 1 \ (\text{mod } 11) )$$）。
    所以 ( $$3^{-1} \equiv 4 \ (\text{mod } 11) $$)。

    (2) 费马小定理（当 ( m ) 是质数时）

    如果 ( m = p ) 为质数，且 ( p \nmid a )，那么：

    $$a^{p-1} \equiv 1 \pmod{p}$$

    因此：

    $$a^{-1} \equiv a^{p-2} \pmod{p}$$

    用快速幂计算即可。

    例子：在模 ( p=7 ) 下，求 ( 3 ) 的逆元：
    ( $$3^{7-2} = 3^5 = 243 $$)，
    ( $$243 \div 7 = 34 \times 7 = 238$$)，余 ( 5 )，
    所以 ( $$3^{-1} \equiv 5 \ (\text{mod } 7)$$ )。
    验证：( $$3 \times 5 = 15 \equiv 1 \ (\text{mod } 7) $$)。

    (3) 欧拉定理（推广到任意模数，但需知 ($$\phi(m))$$）

    若 (gcd(a,m)=1)，则：

    $$a^{\phi(m)} \equiv 1 \pmod{m}$$

    其中 ($$\phi(m)$$) 是欧拉函数。
    于是：

    $$a^{-1} \equiv a^{\phi(m)-1} \pmod{m}$$

    这个方法在 ( $$\phi(m) $$) 容易计算时才方便。

    一、题意重述

    要求解同余方程： $$a x \equiv 1 \pmod b$$

    也就是说： $$a x - 1 = k b$$

    其中 (x) 是我们要找的最小正整数解。 x 是 a 在模 b 意义下的乘法逆元

    二、思路分析

    1️⃣ 暴力思路（只适合小数据）

    • 枚举$$ (x = 1 \sim b)$$
    • 判断 (a*x - 1) % b == 0
    • 时间复杂度：(O(b))

    在 b ≤ 1000 时还能活， 但 b 最大到 2×10⁹，暴力直接“阵亡”。

    2️⃣ 正解：扩展欧几里得（核心思想）

    🔹 数学转化

    原式：

    $$a x \equiv 1 \pmod b$$

    等价于：

    $$a x + b y = 1$$

    这已经是扩展欧几里得算法的标准形式。

    🔹 什么时候有解？

    当且仅当：

    $$\gcd(a, b) = 1$$

    保证一定有解，可以放心算。

    🔹 扩欧给了我们什么？

    扩展欧几里得能找到一组整数解：

    $$a x_0 + b y_0 = 1$$

    那么：

    • (x_0) 就是一个解
    • 但可能是负的，需要调整到最小正整数

    🔹 如何变成最小正整数解？

    所有解的形式是：

    $$x = x_0 + k b$$

    我们要把它调整到：

    $$x \in [1, b]$$

    常用写法：

    x = (x % b + b) % b;

    如果结果是 0，要换成 b。

    三、参考代码

    #include <iostream>
using namespace std;

long long a, b;

/*
  扩展欧几里得算法
  求 ax + by = gcd(a, b) 的一组解
*/
long long exgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;  // gcd(a, 0) = a
    }
    long long x1, y1;
    long long g = exgcd(b, a % b, x1, y1);

    // 由递推关系回推
    x = y1;
    y = x1 - (a / b) * y1;
    return g;
}

int main() {
    cin >> a >> b;

    long long x, y;
    exgcd(a, b, x, y);  // 得到 ax + by = 1

    // 将 x 调整到 [1, b] 范围内的最小正整数解
    x = (x % b + b) % b;
    if (x == 0) x = b;

    cout << x << endl;
    return 0;
}

    //暴力枚举
#include <iostream>
using namespace std;

int main() {
    long long a, b;
    cin >> a >> b;
    // 从 1 枚举到 b，寻找最小正整数解
    for (long long x = 1; x <= b; x++) {
        // 判断 (a*x - 1) 是否能被 b 整除
        if ((a * x - 1) % b == 0) {
            cout << x << endl;
            return 0;  // 找到最小解，直接结束
        }
    }
    // 按题意一般保证有解，这里只是兜底
    return 0;
}
